Stack Exchange


Before getting into any programs or coding languages, you should be introduced to the best educational resource on the planet: Stack Exchange. Its value cannot be understated. The site is simply a Q&A forum for certain topics. For instance, Stack Overflow is the site for programming questions. There is also one for Math questions , and if you're so inclined, even one for cooking questions (aptly named, Seasoned Advice). The community that has developed around this site is knowledgable and deep. If you ever have a programming question on syntax or method, google it with "stack overflow" to get some excellent answers. Anyone that has spent time programming (and I'm including R) will corroborate that this is an indispensible service.

Microsoft Excel


Overview

Excel is a spreadsheet-based software. Unlike SQL and R (and all programming languages), it is not command-based but rather interface-based. This means that rather than writing out a line of code that says "take A + B", you open a spreadsheet and interact with a menu bar and other commonplace functionalities of a graphic interface. Excel is used everywhere, and its utility should not be overlooked.

MySQL


Overview

SQL, or Structured Query Language, is a programming language developed to store (and then access) data in a relational database format. It's extremely common to organizations that collect their own data. Take the Stanford Bookstore website, as an example. The catalog of items displayed on the page is surely referencing a table in a relational database that says which items are in- or out of- stock. And if you login, then your email likely also gets recorded in another table for site visitors. This type of data storage is what would allow you to "save your shopping cart" and come back to it later. And for the data analyst, SQL is the language used to access this stored data.

While the Stanford Bookstore example probably doesn't resonate with your career, I'm including SQL on this site for two reasons. First, even if you aren't accessing data stored in a "SQL server", you can still set one up to store your own data and utilize the simplicity of SQL to manipulate your data. Second, understanding SQL is a great foundation for manipulating data in R, Excel, and even Python or other languages. In fact, these all borrow heavily from SQL. R's "dplyr" package uses SQL-style syntax; Excel's pivot tables are a visual implementation of SQL code.

It's important to note that there are a few different versions of SQL. "MySQL" is one of them. Others include "SQL Server" and "PostgreSQL". Each will have some subtle differences; keep this in mind as you are looking at Stack Overflow forums or googling "help with sql code ___". For this site, all SQL references are effectively in MySQL.

R


Overview

R is a statistical programming language. By now, you've most likely come into contact with it through the IPS, PoliSci, or Econ curriculums. Its open-source nature has allowed it to develop extensively over the years, and remains a favorite for academics and professionals. You may have found that Python is widely used in data analysis and data science circles. This is true. However, unless you intend to be a data engineer, R will suit you well and -- like Excel and SQL -- you will find an extremely wide base of people also using R.

Installation

Remember to draw the distinction between "R" and "RStudio". The former is the programming language, and the latter is an IDE (integrated development environment). With a bad analogy, R is the airplane and Rstudio is the airport. You cannot run RStudio without first installing R. I would fully recommend using RStudio for all of your projects.
