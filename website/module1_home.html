Module 1
Data Manipulation



The purpose of this module is to answer common data questions in each of three programs: (1) Excel, (2) SQL, (3) R. Although you will find that R is the most flexible among these three, Excel is the common denominator in most organizations and so its power, simplicity, and ubiquity cannot be overlooked. SQL is included as most organizations with their own data will store it in a relational database, for which SQL is used to extract data. And R is your power toolkit for nearly limitless data analysis.

No modeling or statistics are employed here. This module simply demonstrates the manipulation of data into aggregations and trends (which you may find is not so simple, in fact).

For some on the surface, this can help you understand the steps that a data analyst in your organization would take; it may also show you how to query a database or answer rudimentary questions yourself. For others that wish to go deeper, mastering these techniques is the absolute baseline requirement to being a proper data analyst.

By the end of Module 1, you should be able to do the following:

  • Utilize Excel for basic e.g. =AVERAGE(), intermediate e.g. =VLOOKUP(), and advanced e.g. =INDEX(MATCH()) formulas, including multiple nested formulas.
  • Build Excel pivot tables from normalized data, and understand the difference between normalized and non-normalized data
  • Connect to a SQL database and run basic queries
  • Utilize R's dplyr package for aggregating data
  • Know the advantages and limitation across Excel, SQL, and R
  • Learn something about national GDPs, Rural Populations, or Health Expenditure statistics
