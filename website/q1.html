Question 1

Math and Statistics Calculations

The purpose of this question is to demonstrate these programs' utility as calculators. All scientific programming languages -- e.g. MATLAB, python, and many others -- are going to have these functionalities built in. From as simple as 1 + 1 = 2 to as challenging as a correlation formula. Each program has its own syntax, and this is a great place to start to unfold it and learn it. At the end of this question, you should have a general sense of what operations are easy, and which are challenging, across Microsoft's Excel, R, and Oracle's MySQL.

A: What is the average annual GDP per capita of Argentina from 2005 through 2016?


Solutions (click to expand)

Excel


  // Use the average formula over your cells of interest

  =AVERAGE(GDP.PCAP!AZ9:BK9)

  // Remember that all functions have some default error-handling. This is how the function accounts for bad inputs
  // In Excel, run the following to see
  =AVERAGE("not a number")

SQL


  // MySQL also has an average function

  select avg(gdp_pcap)
  from gdp_pcap
  where region_name = 'Argentina'
  and year between 2005 and 2016

  // Error handling for avg in SQL:
  select gdp_pcap from gdp_pcap where region_name = 'Argentina' and year = 1961 // this is a 'NULL' value
  select avg(gdp_pcap) from gdp_pcap where region_name = 'Argentina' and year = 1961 // try to take the avg of a NULL

R

 
  // In R, this is the 'mean' function

  set = gdp$region_name == 'Argentina' & gdp$year >= 2005 & gdp$year <= 2016
  mean( gdp$gdp_pcap[ set ] )

  // Error handling for mean in R
  // Remember, you can always google about a function to learn how it handles errors. In Rstudio, you can also type ?[function]
  test = c(4,8,NA,12)
  mean(test)
  mean(test, na.rm=T) // specify how to handle NAs. Aside: what's the difference between NULL and NA in R?

E2: What is the average % of Public Expenditure on health in 2014, weighted by Rural Population in the same year?


Solutions (click to expand)

Excel


  // A weighted average can be accomplished with the SUMPRODUCT function
  // this multiplies the 1st and 2nd arguments you pass to it, and then takes their sum
  // generically: sum(value * weights) / sum(value) 

  =SUMPRODUCT(HEALTH!BH2:BH265,RUR.POP!BH2:BH265)/SUM(RUR.POP!BH2:BH265)

SQL


  // SQL doesn't have an explicit sum-product function, but it's trivial to take then sum of the product
  // if this is your first time joining, make sure you understand what's going on by taking a SQL tutorial

  select sum(rp.rural_pop * he.health_exp)/sum(rp.rural_pop)
  from rural_pop rp
  join health_exp he 
    on he.region_code = rp.region_code 
    and he.year = rp.year
  where rp.year = 2014

R

 

  // Again, just taking utilizing the same formula: sum(value * weights) / sum(value) 

  sum(health$health_exp[health$year==2014] * pop$rural_pop[pop$year==2014], na.rm=T ) / 
    sum( pop$rural_pop[pop$year==2014], na.rm=T ) 

  // but wait! the above only works if the tables 'health' and 'pop' have all countries in the same order. We shouldn't presume the do:

  // first, merge the two tables accordingly
  new = merge(health, pop, by=c("region_code", "year"))

  // hmmm, many of the columns got duplicated, that's messy...
  names(new)

  // try again, now specifying only the new columns you want from the population table
  new = merge(health, pop[,c("region_code","year","rural_pop")], by=c("region_code", "year"))

  // and calculate the weighted average now
  sum(new$health_exp[new$year==2014] * new$rural_pop[new$year==2014], na.rm=T ) / 
    sum( new$rural_pop[new$year==2014], na.rm=T )

H: What is the correlation between Rural Population and % spent on healthcare in the following countries for 2000-2014?
Zimbabwe, Sweden, Philippines


Solutions (click to expand)

Excel


  =CORREL(HEALTH!AT265:BH265,RUR.POP!AT265:BH265) // Zimbabwe
  =CORREL(HEALTH!AT223:BH223,RUR.POP!AT223:BH223) // Sweden
  =CORREL(HEALTH!AT187:BH187,RUR.POP!AT187:BH187) // Philippines

SQL


  // Just like median in SQL, this is not Easy...but here it is. I'd recommend only going through this once you're comfortable with advanced SQL queries.
  // MySQL does not have a correlation function (like it does for average, or standard deviation)
  // Remember, sample correlation coefficient equation: https://wikimedia.org/api/rest_v1/media/math/render/svg/bd1ccc2979b0fd1c1aec96e386f686ae874f9ec0

  select region_name, p1 / (sqrt(p2)*sqrt(p3)) as correl_coef
  from (
  select 
    region_name, 
    sum((x-xbar)*(y-ybar)) as p1,
    sum(power(x-xbar,2)) as p2,
    sum(power(y-ybar,2)) as p3
  from (
  select 
    rp.region_name, 
    rp.rural_pop as x, 
    x.avg_pop as xbar, 
    he.health_exp as y, 
    x.avg_health as ybar
  from rural_pop rp
  join health_exp he on he.region_code = rp.region_code and he.year = rp.year
  join (
    select 
      rp.region_name,
      avg(rp.rural_pop) as avg_pop, 
      avg(he.health_exp) as avg_health 
    from rural_pop rp
    join health_exp he 
      on he.region_code = rp.region_code 
      and he.year = rp.year
    where rp.region_name in ('Zimbabwe', 'Sweden', 'Philippines')
    and rp.year between 2000 and 2014
    group by rp.region_name
  ) x on x.region_name = rp.region_name
  where rp.region_name in ('Zimbabwe', 'Sweden', 'Philippines')
  and rp.year between 2000 and 2014
  order by rp.region_name
  ) y
  group by region_name
  ) z

R



  // Fortuntely, R does have a correlation function

  new = merge( health[ ,c("region_name","year","health_exp") ], 
               pop[ ,c("region_name","year","rural_pop") ], 
               by = c("region_name", "year") 
  )
  new = new[new$region_name %in% c('Zimbabwe', 'Sweden', 'Philippines'), ]   # only certain countries 
  new = new[new$year >= 2000 & new$year <= 2014, ]                           # only certain years
  new = new[ is.na(new$health_exp)==FALSE & is.na(new$rural_pop)==FALSE, ]   # only full records
  cor(new$rural_pop, new$health_exp)

I: What is the correlation between Rural Population and % spent on healthcare over all countries for 2000-2014?


Solutions (click to expand)

Excel


  // Usefully, you can pass multiple vectors (i.e. a string of years for a country and the health expenditure values) into the single function 

  =CORREL(HEALTH!AT2:BH265,RUR.POP!AT2:BH265)

SQL


  // Similar as the last SQL answer. Need to implement on our own the correlation formula

  select p1 / (sqrt(p2)*sqrt(p3)) as correl_coef
  from (
  select 
    sum((x-xbar)*(y-ybar)) as p1,
    sum(power(x-xbar,2)) as p2,
    sum(power(y-ybar,2)) as p3
  from (
  select 
    rp.rural_pop as x, 
    x.avg_pop as xbar, 
    he.health_exp as y, 
    x.avg_health as ybar
  from rural_pop rp
  join health_exp he on he.region_code = rp.region_code and he.year = rp.year
  cross join (
    select 
      avg(rp.rural_pop) as avg_pop, 
      avg(he.health_exp) as avg_health 
    from rural_pop rp
    join health_exp he 
      on he.region_code = rp.region_code 
      and he.year = rp.year
    where rp.year between 2000 and 2014
  ) x 
  where rp.year between 2000 and 2014
  ) y
  ) z

R

 
  // Almost identical to the prior R answer, except removing one line. Do you see it?

  new = merge( health[ ,c("region_name","year","health_exp") ], 
               pop[ ,c("region_name","year","rural_pop") ], 
               by = c("region_name", "year") 
  )
  new = new[new$year >= 2000 & new$year <= 2014, ]                           # only certain years
  new = new[ is.na(new$health_exp)==FALSE & is.na(new$rural_pop)==FALSE, ]   # only full records
  cor(new$rural_pop, new$health_exp)
